When you speak to the cluster you connect to the login node. Behind teh login node there is the master node, which is hiden for us (users). On the same level
we have a bunch of computer nodes, about a 100 nodes, in between these compute nodes, there is a fast network, a fast ethernet connection and behind 
there are also storage nodes, which all the harddrives. All data lives there but as a user you don't see it. All you interested resides in the computer nodes. 


SLURM is the language to use and distribute them between the computers. SLURM is probably used in 90% of all hpc systems, SOTA for job scheduling. 


sinfo --long --Node --> displays the nodelist. 

All the cluster is isplitted in different partitions. There is a partition named GPU, these are all the nodes that contain GPU, parititon 
all contains CPU, it makes sense to partition these nodes, because there might be jobs with CPU intensive and CPU intensive. 

Partition mig-preempt and mig-priority (we don't really know what they do, runned by admin). Long are jobs that run for weeks. 

State, this tells which kind of state is this node: Mixed tells us this nodes is taking jobs but still there is some free resources to use. 
Then we have allocated nodes, these are full, they can't accept any new jobs. If a node is completely free it would say idle. 

In the CPUS column, we can see how many CPU cores are available in each node. 
S:C:T column, tell how are these cores configured in a compute nodes. The first number tells the number of sockets, how many cores per socket and how many 
threats can each core in parallel. The product gives the number of cpu's available. The memory is in MB, so 256GB of raw available. 

TMP_disk is some temporary storages connected to a node. 
weigth is used for teh scheduling system in order to priorityze eahc node 

column is available features: you can see some features assigned to each node. 

Last colum is reason: none everywhere, which means that all the nodes are online currently.

If we put 256, does it take a lot of time? we need CPU, memory, time   

For every task in your job, we need a core of a cpu. How much memory did I get: more cpu's you request more memory you will get, you can 
request more memory for cpu, the default value is 2GB per CPU. In that case, I request 512 GB of ram from teh cluster. How much time did i get 
to run this job? By default you get half an hour. There is an upper limit on how much time you can request depending on the partition. on the GPU is 4 days 
and in the long partition is 14 days. The time starts when the allocation is complete. 

^c cancels your job allocation. 

You can also explicitly request on how many jobs your nodes should run: 
srun --ntasks=4 --nodes=2 hostname 

Usually it distributes them equally among the nodes.
