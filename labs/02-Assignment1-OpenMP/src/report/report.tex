\begin{enumerate}[label={\textbf{Exercise \arabic*:}}]
  \item We parallelize all algorithm steps with OpenMP, specifically using the basic parallel approach for cumulative energy and parallelizing only the copying phase of seam removal. All code is compiled with \texttt{-O3} optimizations. 

Execution times are measured across five image resolutions ($720 \times 480$ to $7680 \times 4320$) using 1, 2, 4, 8, 16, and 32 threads. Each test case requires at least 5 runs removing 128 seams to calculate average timings. Performance is evaluated via speed-up, defined as $S = \frac{t_s}{t_p}$, where $t_s$ is the sequential runtime and $t_p$ is the parallel runtime.


  \item We implement greedy seam removal to enable the parallel processing of multiple seams and we analyze the trade-offs between execution speed and final image quality. For each algorithm step and image resolution, we perform a scaling study to determine the optimal thread and core counts.
  
\begin{enumerate}
  \item   We proceed now to implement triangle-based cumulative energy approach. We compared two tiling strategies where the image is divided into alternating upward and downward triangular blocks.
  \begin{enumerate}
      \item \textbf{Case 1 ($Y \pm 1$):} The tiles have a horizontal tip of two blocks. This approach present the pink dots not uniformly distributed. 
      
      \item \textbf{Case 2 ($Y \pm 2$):} The tiles have a single-block tip. This approach present the pink dots uniformly distributed. 
  \end{enumerate}

  Computationally, both options present similar performance. However, in this study the second approach (single-block tip) seems more suitable because when the height of the triangle is $k=1$, we fully return to the standard sequential approach, and, hence, we have a robust baseline to compare.

\item To optimize the seam selection process, we modified the aforementioned traditional dynamic programming approach by implementing a bidirectional energy accumulation. 

The image is bisected at $\bar{y} = \lceil h/2 \rceil$, we execute the parallel triangular approach in parallel, on starting on the top and the other on the bottom as shown in image \ref{fig:midpoint_sync}. The final seam is reconstructed by identifying the optimal midpoint $\hat{x}$ via the formula 
\begin{equation}
  \hat{x} = \underset{i}{\operatorname{argmin}} \left( E(x_i, \bar{y}) + \min \left( E(x_{i-1}, \bar{y}), E(x_i, \bar{y}), E(x_{i+1}, \bar{y}) \right) \right)
\end{equation}
and performing two simultaneous tracebacks, one moving upward and one moving downward from the select $\hat{x_i}$.
